## System Overview

VoxSwarm v3.2.0 is a **multi-agent opinion simulation engine** for Indonesian public policy scenarios. It uses LLM-powered agents to simulate stakeholder debate, then optionally runs ML models for prediction.

**Core flow:**
```
Input (scenario) → Agent Registry → Real Context (scraper) → Simulation Loop → Response Formatting
                                              ↓                                           ↓
                                        Sentiment Analysis                          Metrics + Reporting
                                              ↓                                           ↓
                                        ML Pipeline (optional)                   Feedback Loop
```

**Two tiers:**
| Feature | Free | Normal |
|---------|------|--------|
| Agents | 4 max | 7+ max |
| Sentiment | Inline keyword | ML/LLM |
| ML Prediction | ❌ | ✅ |
| Analysis | ❌ | ✅ |
| Reports | ❌ | ✅ |

## File-by-File Breakdown

### Entry Point

#### `main.py` (897 lines)
- **Purpose:** FastAPI app, 18 API endpoints, CORS, rate limiting, startup scheduler
- **LLM calls:** None directly
- **Dependencies:** `backend.simulation`, `backend.feedback`, `backend.scraper`, `backend.ml_pipeline`, `backend.metrics`
- **Endpoints:**
  - `POST /simulate` — single scenario (free/normal)
  - `POST /simulate/batch` — multi-scenario batch
  - `GET /simulate/{id}/explain` — explainability report
  - `POST /feedback` — ground truth submission
  - `POST /feedback/{id}/override-label` — manual label correction
  - `GET /feedback/stats` — feedback statistics
  - `POST /feedback/retrain` — force ML retrain
  - `GET /metrics/prediction` — latest ML metrics
  - `GET /metrics/ml-pipeline` — full pipeline metrics
  - `GET /metrics/ground-truth` — ground truth stats
  - `GET /context/cache-stats` — scraper cache stats
  - `POST /context/clear-cache` — clear scraper cache
  - `POST /background-simulation` — trigger background sim
  - `GET /scheduler/status` — scheduler state
  - `POST /scheduler/pause` — pause scheduler
  - `POST /scheduler/resume` — resume scheduler
  - `POST /scheduler/trigger` — manual trigger
  - `GET /health` — health check
- **Notes:** Rate limiter is in-memory only (resets on restart). Scheduler manages background sims.

### Core Simulation

#### `backend/simulation.py` (1267 lines) ⭐ CRITICAL
- **Purpose:** Main orchestrator — runs multi-round agent debate
- **LLM calls:**
  - `generate_response()` — per agent per round (up to N agents × M rounds)
  - `generate_qa_check()` — 1 per round (Normal tier only)
  - `analyze_scenario()` — 1 per simulation (Normal tier only)
  - `identify_key_actors()` — 1 per simulation (Normal tier only)
- **Key functions:**
  - `run_simulation()` — main entry point
  - `_run_multi_round_simulation()` — core loop
  - `_get_dynamic_round_count()` — adjusts rounds based on complexity
  - `_get_round_context()` — builds context for each round
  - `_apply_event_effects()` — modifies agents based on injected events
  - `_check_consensus()` — detects early stopping
  - `_cap_opinion_changes()` — prevents radical stance swings
  - `_generate_final_summary()` — wraps up simulation
- **Dependencies:** `agents`, `memory`, `llm`, `sentiment`, `event_system`, `scheduler`, `swarm`, `metrics`, `graph`
- **Notes:** This is the biggest file. Most complexity lives here.

#### `backend/agents.py` (259 lines) ⭐ CRITICAL
- **Purpose:** Agent profile registry — 9 default Indonesian stakeholder archetypes
- **Profiles:**
  1. Mahasiswa (Student) — neutral, high engagement
  2. Pengusaha/UMKM (Business) — profit-focused
  3. Pekerja Kantoran (Office Worker) — stability-focused
  4. Pemerintah (Government) — policy-focused, authoritative
  5. Akademisi (Academic) — data-driven, cautious
  6. Oposisi Kritis (Opposition) — critical, skeptical
  7. Jurnalis/Media (Journalist) — balanced, investigative
  8. Masyarakat Umum (General Public) — apathetic, self-interest
  9. Tokoh Agama (Religious Leader) — moral authority
- **Constants:**
  - `KATEGORI_MAP` — keyword→category mapping for injection
  - `COUNTER_MAP` — counter-narrative responses per category
- **LLM calls:** None
- **Dependencies:** None (pure data)
- **Notes:** Core to system. Each profile has `skor awal`, `volatilitas`, `susceptibility`.

#### `backend/agent_factory.py` (203 lines)
- **Purpose:** Dynamic agent injection based on scenario keywords (archetype-based)
- **Archetypes:**
  1. Perwira_TNI — military perspective
  2. Dokter_Nakes — healthcare worker
  3. Petani — farmer
  4. Nelayan — fisherman
  5. Pemuda_Digital — digital youth
  6. Guru_Dasar — elementary teacher
- **LLM calls:** None
- **Dependencies:** `agents.KATEGORI_MAP`, `agents.COUNTER_MAP`
- **Notes:** Injects additional agents when keywords match. Good for specificity.

### Memory & Context

#### `backend/memory.py` (177 lines)
- **Purpose:** Agent memory management — tracks past opinions for context
- **LLM calls:** None (but previously had LLM summarization, now removed)
- **Key functions:**
  - `build_memory_context()` — formats memory into prompt text
  - `extract_stance_from_memory()` — gets latest stance label
  - `update_memory()` — adds new round's opinion to memory
- **Dependencies:** `core.memory_store` (uses structured memory if available)
- **Notes:** Simple dict-based + optional structured memory overlay.

#### `backend/core/memory_store.py` (302 lines)
- **Purpose:** Structured memory — argument uniqueness tracking, relationship tracking
- **Classes:**
  - `ArgumentMemory` — tracks unique arguments, detects repetition
  - `RelationshipMemory` — tracks alignment/trust between agents
  - `AgentMemoryStore` — combined store per agent
- **LLM calls:** None (pure rule-based)
- **Key features:**
  - Keyword extraction via regex (no LLM needed)
  - Argument similarity via Jaccard on extracted terms
  - Trust summary from sentiment alignment
- **Dependencies:** None
- **Notes:** Good quality, zero cost. Keeps agents from repeating themselves.

### Sentiment Analysis

#### `backend/sentiment.py` (167 lines)
- **Purpose:** Multi-mode sentiment analysis (3 modes)
- **Modes:**
  1. `inline` — keyword dictionary (free)
  2. `ml` — scikit-learn classifier (free, fast)
  3. `llm` — Groq LLM call (paid)
- **LLM calls:** Only if mode="llm" (1 call per invocation)
- **Key functions:**
  - `analyze_sentiment()` — main entry, dispatches to mode
  - `_inline_sentiment()` — keyword matching
  - `_ml_sentiment()` — uses `sentiment_ml.py`
  - `_llm_sentiment()` — calls `llm.py`
- **Dependencies:** `sentiment_ml`, `llm`
- **Notes:** Free tier uses inline. Normal tier uses ml or llm.

#### `backend/sentiment_ml.py` (128 lines)
- **Purpose:** ML-based sentiment classifier (TF-IDF + LogisticRegression)
- **LLM calls:** None
- **Key features:**
  - Bootstrap training dataset (Indonesian policy terms)
  - Auto-trains on first use if no model exists
  - Saves model to `models/sentiment_model.pkl`
- **Dependencies:** `sklearn` (LogisticRegression, TfidfVectorizer)
- **Notes:** Lightweight, fast. Good enough for policy opinions.

### ML Pipeline (Prediction)

#### `backend/ml_pipeline.py` (205 lines)
- **Purpose:** Feature extraction + RandomForest prediction model
- **Features extracted (22 total):**
  - Agent sentiment features (mean, std, min, max, range)
  - Agent stance features (proportion, volatility)
  - Event features (count, sentiment impact)
  - Context features (word count, entity count, category count)
  - Memory features (depth, freshness, argument count)
- **LLM calls:** None
- **Key functions:**
  - `extract_features()` — builds feature vector from simulation
  - `train()` — trains RandomForest classifier
  - `predict()` — predicts outcome class + probability
  - `evaluate()` — cross-validation metrics
  - `check_drift()` — detects model degradation
- **Dependencies:** `sklearn` (RandomForestClassifier, cross_val_score), `joblib`
- **Notes:** Experimental. Trained on synthetic data initially. Auto-retrains every 5 simulations.

#### `backend/feedback.py` (189 lines)
- **Purpose:** Ground truth collection, label override, retrain trigger
- **LLM calls:** None
- **Key functions:**
  - `submit_feedback()` — stores ground truth
  - `override_label()` — manual label correction
  - `get_feedback_stats()` — statistics
  - `check_retrain_trigger()` — auto-retrain if threshold=3
  - `force_retrain()` — manual retrain trigger
- **Dependencies:** `ml_pipeline`
- **Notes:** Feedback stored in `backend/data/feedback.jsonl`. Retrain threshold is 3 samples.

- **RSS Feeds:**
  - Kompas (3 feeds)
  - Detik (3 feeds)
  - BBC Indonesia
  - Tempo
  - Antara News
  - CNN Indonesia
  - Tirto
- **LLM calls:** 1 per invocation (GraphRAG entity extraction, ~350 tokens)
- **Key functions:**

  - `_fetch_rss_feeds()` — parallel RSS fetch with thread pool
  - `_extract_entities_with_llm()` — LLM entity extraction

- **Dependencies:** `graph` (GraphRAG), `feedparser`, `sklearn` (TfidfVectorizer)

#### `backend/graph.py` (268 lines)
- **Purpose:** GraphRAG-lite entity/relation extraction (no graph database)
- **LLM calls:** 1 per invocation (350 token budget)
- **Key functions:**
  - `extract_entities_relations()` — main extraction
  - `_build_entity_prompt()` — constructs extraction prompt

- **Dependencies:** `llm`
- **Notes:** Lightweight alternative to full GraphRAG. Entities extracted: Person, Organization, Location, Policy, Event. Relations: supports, opposes, affects, mentions.

### Metrics & Prediction

#### `backend/core/metrics.py` (97 lines)
- **Purpose:** Core opinion metrics — polarization, consensus, volatility, conflict
- **LLM calls:** None (pure math)
- **Functions:**
  - `compute_polarization()` — variance of final stances
  - `compute_consensus()` — 1 - polarization
  - `compute_volatility()` — average absolute stance change per agent
  - `compute_conflict_score()` — proportion of opposing agents × intensity
- **Dependencies:** None
- **Notes:** Simple, correct, no cost.

#### `backend/core/prediction.py` (348 lines)
- **Purpose:** Heuristic prediction engine (primary) + LLM prediction (Normal tier)
- **Prediction approach:**
  1. **Heuristic (primary):** 6-factor weighted model
     - Support ratio
     - Average sentiment
     - Polarization
     - Opposition intensity
     - Momentum (last round change)
     - Volatility
  2. **LLM (optional):** Groq call for prediction
  3. **Ensemble:** Combines heuristic + ML + LLM
- **LLM calls:** 0-1 per prediction (only if Normal tier + LLM enabled)
- **Key functions:**
  - `predict_probability()` — main prediction
  - `_heuristic_prediction()` — 6-factor model
  - `_combine_predictions()` — ensemble logic
  - `_adjust_for_events()` — event impact weighting
  - `_apply_rule_adjustments()` — domain-specific rules
- **Dependencies:** `metrics`
- **Notes:** Heuristic is primary, ML is experimental. LLM prediction is optional.

### Scheduling & Orchestration

#### `backend/core/scheduler.py` (126 lines)
- **Purpose:** Agent speaking order management
- **Modes:**
  1. `influence_aware` — higher influence agents speak first
  2. `sequential` — alphabetical
  3. `randomized` — shuffled
- **LLM calls:** None
- **Key functions:**

  - `select_response_targets()` — who responds to whom

- **Notes:** Simple but effective. influence_aware is default.

#### `backend/core/swarm.py` (260 lines)
- **Purpose:** Rule-based crowd agents (50-200) that react to LLM agents
- **LLM calls:** None
- **Key features:**
  - 7 demographic groups (mahasiswa, pekerja, umkm, masyarakat, akademisi, media, oposisi)
  - Propagation: crowd stance moves toward affinity LLM agent
  - Clustering: pro/netral/kontra groups
  - Propagation graph: LLM → crowd influence visualization
- **Dependencies:** None
- **Notes:** Cheap way to scale opinion diversity. No LLM cost.

#### `backend/core/models.py` (290 lines)

- **Models:**

  - `ExplanationReport`, `HTMLReport`
  - `ComparisonResult`, `MetricSnapshot`
- **Adapters:** `from_legacy_dict()` for backward compatibility
- **LLM calls:** None
- **Dependencies:** `pydantic`
- **Notes:** Clean data contracts. Legacy adapters help with old data.

### Infrastructure

- **Purpose:** Groq LLM client with fallback chain
- **Models (fallback order):**
  1. `llama-3.3-70b-versatile` — primary
  2. `llama-3.1-8b-instant` — fallback 1
  3. `gemma2-9b-it` — fallback 2
  4. `mixtral-8x7b-32768` — fallback 3
  5. `llama3-groq-8b-8192-tool-use-preview` — fallback 4

- **Key features:**
  - In-memory cache (TTL 1 hour)
  - Rate limiter (10 req/sec)
  - Retry with exponential backoff (3 attempts)
  - Token budget enforcement
- **Dependencies:** `groq` SDK
- **Notes:** Single point of failure. Cache helps with repeated prompts.

## Removal Recommendations

### KEEP (Essential)
| Component | Why |
|-----------|-----|
| `simulation.py` | Core engine — cannot function without |
| `agents.py` | Agent profiles — cannot function without |
| `llm.py` | LLM client — required for agent responses |
| `memory.py` | Agent memory — prevents context loss |
| `core/memory_store.py` | Structured memory — prevents repetition |

| `core/scheduler.py` | Speaking order — simple, no cost |
| `core/metrics.py` | Metrics — pure math, no cost |
| `main.py` | Entry point — required |

### KEEP (High Value)
| Component | Why | Could Remove If... |
|-----------|-----|---------------------|
| `core/event_system.py` | Adds realism via events | You want simpler sims |
| `core/swarm.py` | Scalable crowd (0 LLM cost) | You only need 4-9 agents |
| `core/prediction.py` | Heuristic prediction (primary) | You skip prediction |
| `sentiment.py` | Required for stance tracking | Never |
| `agent_factory.py` | Dynamic agent injection | You want fixed agent set |

### KEEP (Low Value / Experimental)
| Component | Why | Could Remove If... |
|-----------|-----|---------------------|
| `ml_pipeline.py` | Experimental ML prediction | You don't use ML predictions |
| `feedback.py` | Ground truth collection | You don't do supervised learning |
| `scraper.py` | Real-world context | You want pure simulation |
| `graph.py` | Entity extraction | You simplify context |
| `sentiment_ml.py` | ML sentiment | You only use inline |
| `core/reporting.py` | Explainability reports | You don't need explanations |
| `core/comparison.py` | Cross-scenario comparison | You only run single scenarios |

### REMOVE (If Simplifying)
| Component | Why | Replacement |
|-----------|-----|-------------|
| `scraper.py` + `graph.py` | Complex, LLM cost per fetch | Hardcode context or skip |
| `ml_pipeline.py` + `feedback.py` | Experimental, needs training data | Use heuristic only |
| `core/comparison.py` | Niche use case | Manual comparison |
| `core/reporting.py` | Nice-to-have | Return raw JSON |

**Optional (add back as needed):**

- `core/swarm.py` — for crowd scale
- `core/prediction.py` — for predictions

**Removable (for simplification):**
- `scraper.py` + `graph.py` — context fetch
- `ml_pipeline.py` + `feedback.py` — ML prediction
- `core/reporting.py` + `core/comparison.py` — analysis
